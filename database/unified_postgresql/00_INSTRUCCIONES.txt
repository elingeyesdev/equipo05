================================================================================
BASE UNIFICADA PostgreSQL — equipo05_unificado
Carpeta: database/unified_postgresql/
================================================================================

0) DOCKER DESKTOP (recomendado si PostgreSQL en Windows te da error 1067)
   En la raiz del repo (equipo05-main-integration):
     - Opcional: en .env define UNIFIED_PG_PASSWORD y UNIFIED_PG_DOCKER_PORT (default 5433).
     - Levantar solo la base unificada:
         docker compose up -d db_unificado
     - La primera vez crea el volumen y ejecuta automaticamente los SQL 01..08
       (archivo database/docker/00-init-unificado-db.sql; sin bash para evitar CRLF en Windows).
     - pgAdmin / DBeaver / Laravel: database/docker/CONEXION_DBEAVER_Y_LARAVEL.txt
     - Si cambiaste scripts SQL y queres recrear desde cero (solo desarrollo):
         docker compose stop db_unificado
         docker volume rm equipo05-main-integration_pg-unificado-data
         (el nombre del volumen puede variar; miralo con: docker volume ls)
         docker compose up -d db_unificado

1) PGADMIN / DBeaver — CONECTAR AL SERVIDOR
   - Host localhost (o tu Docker), puerto 5432, usuario/contrasena PostgreSQL.

2) CREAR LA BASE (una sola vez)
   - Host localhost (o tu Docker), puerto 5432, usuario/contrasena PostgreSQL.

2) CREAR LA BASE (una sola vez)
   - Conectado a la base "postgres", ejecutar SOLO:
     00_create_database.sql
   - Desarrollo: DROP DATABASE equipo05_unificado; (cerrar conexiones antes).

3) ESQUEMAS Y TABLAS (orden recomendado)
   Opcion A — Script automatico (requiere psql en PATH):
     PowerShell (Windows), desde esta carpeta:
       $env:PGPASSWORD = "tu_clave"
       .\run_schema_all.ps1
     Bash (Linux/macOS):
       export PGPASSWORD=tu_clave
       chmod +x run_schema_all.sh
       ./run_schema_all.sh

   Opcion B — Manual en DBeaver/pgAdmin, conexion a "equipo05_unificado":
     01_extensions_and_schemas.sql
     04_mod_inventario_transparencia.sql
     02_mod_incendios.sql
     03_mod_rescate.sql
     06_mod_logistica.sql
     07_mod_seguimiento.sql
     08_mod_cuadrillas.sql

4) LARAVEL — PERSISTIR MODULOS EN ESTE MISMO PostgreSQL
   En .env del proyecto de integracion:
     DATABASE_UNIFIED_POSTGRES=true
     UNIFIED_PG_HOST=127.0.0.1
     UNIFIED_PG_PORT=5433
     (5433 = puerto publicado por Docker db_unificado; si usas PostgreSQL instalado
      en Windows sin Docker, suele ser 5432.)
     UNIFIED_PG_DATABASE=equipo05_unificado
     UNIFIED_PG_USERNAME=postgres
     UNIFIED_PG_PASSWORD=tu_clave
   La conexion default (DB_CONNECTION) puede seguir en sqlite para la app
   principal (donaciones / sesiones) o migrarla a pgsql si queres todo en PG.

5) DIAGRAMA (ERD) EN DBeaver
   - Conexion a "equipo05_unificado".
   - Clic derecho en un esquema (inventario, incendios, rescate, logistica,
     seguimiento, cuadrillas) -> View Diagram.

6) REPOSITORIO UNIFICADO (GitHub)
   https://github.com/elingeyesdev/equipo05.git

   git clone https://github.com/elingeyesdev/equipo05.git equipo05-main-integration

7) LOS 7 "ESPACIOS" DE DATOS
   - Seis modulos = seis schemas PostgreSQL arriba.
   - Septimo = aplicacion principal (usuarios globales, donaciones fase 1,
     etc.) en database.sqlite o en schema public / core segun configures
     DB_CONNECTION en .env.
